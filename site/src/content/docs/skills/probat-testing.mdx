Cobertura completa do framework ProBat — o framework oficial da TOTVS para testes automatizados no Protheus. Inclui padroes, assertions, templates por tipo de teste e anti-padroes a evitar.

## Quando ativa

Ativa automaticamente quando voce pede para **criar testes unitarios, testes de integracao, testes ProBat** ou testes automatizados para codigo ADVPL/TLPP no Protheus.

## Exemplos de uso

- "Cria testes unitarios para essa funcao de calculo"
- "Preciso de um teste de integracao com setup/teardown"
- "Como testar um endpoint REST com ProBat?"
- "Cria teste para validar o Model MVC"
- "Quais assertions o ProBat oferece?"

## O que oferece

### Estrutura obrigatoria
Todo teste ProBat deve ter: `#include "Protheus.ch"`, User Function retornando oResult, FWTestSuite():New(), testes registrados com AddTest(), e oResult:Run() antes do Return.

### 6 assertions disponiveis
- `IsEqual(valor, esperado, msg)` — compara valores
- `IsTrue(expr, msg)` — valida condicao verdadeira
- `IsFalse(expr, msg)` — valida condicao falsa
- `IsNil(valor, msg)` — verifica retorno nulo
- `IsNotNil(valor, msg)` — verifica que retornou algo
- `HasError(bloco, msg)` — testa tratamento de excecao

### Padrao Setup/Teardown
Dados de teste com prefixo `ZZZ_TEST_`, RecLock para criacao, TCSqlExec DELETE para limpeza. Teardown antes E depois de cada teste.

### 6 templates por tipo de teste
Funcao pura, acesso a dados, endpoint REST, Job/Worker, Model MVC e integracao multi-modulo.

### 8 padroes obrigatorios
Testes independentes, Setup/Teardown, nunca testar UI, prefixo ZZZ_TEST_, limpeza via SQL, caminho feliz e de erro, mensagens descritivas, um assert por conceito.

### 7 anti-padroes
Dependencia de ordem, dados permanentes no banco, teste de interface grafica, assert sem mensagem, multiplos conceitos em um teste, dados sem prefixo, e dependencia de dados existentes.
